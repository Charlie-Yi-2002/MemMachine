## Basic Usage

<Tabs>
  <Tab title="Python SDK">
    ### Initialize MemMachine
```Python
import asyncio
from datetime import datetime

from memmachine.episodic_memory.data_types import ContentType
from memmachine.episodic_memory.episodic_memory_manager import EpisodicMemoryManager

config_path = "config.yaml"

episodic_memory_manager = EpisodicMemoryManager.create_episodic_memory_manager(
    config_path
)

user_id = "my_user"
users = [user_id]
group_id = "my_group"
session_id = "my_session"

episodic_memory = await episodic_memory_manager.get_episodic_memory_instance(
    group_id=group_id,
    session_id=session_id,
    user_id=users,
    config_path=config_path,
)
```
### Store a Memory
    ```python
    await episodic_memory.add_memory_episode(
        producer=user_id,
        produced_for=user_id,
        episode_content="My favorite color is blue.",
        episode_type="default",
        content_type=ContentType.STRING,
        timestamp=datetime.now(),
    )
    ```

### Retrieve Memories/Search Memories
```python
    formalized_query = await episodic_memory.formalize_query_with_context("What is my favorite color?")
    print(f"Formalized query: {formalized_query}")
```

### Add a Memory 

```Python
import requests
import json

# The API endpoint to add memory
url = 'https://memmachine.mintlify.app/llms/add_memmachine'

# The "Hello World" text we want to add
payload = {
    'content': 'Hello, MemMachine! This is my first memory.'
}

# Set up the headers
headers = {
    'Content-Type': 'application/json'
}

# Make the POST request
response = requests.post(url, headers=headers, data=json.dumps(payload))

# Print the response from the server
print(f"Status Code: {response.status_code}")
print(f"Response: {response.text}")
```

### Search for a Memory
```Python
import requests
import json

# The API endpoint to search memory
url = 'https://memmachine.mintlify.app/llms/search_memmachine'

# The query to find our "Hello World" entry
payload = {
    'query': 'first memory'
}

# Set up the headers
headers = {
    'Content-Type': 'application/json'
}

# Make the POST request
response = requests.post(url, headers=headers, data=json.dumps(payload))

# Print the search results
print(f"Status Code: {response.status_code}")
print(f"Search Results: {response.text}")
```

  <Tab title="REST API">
    
This guide provides simple "Hello World" examples for each of the core REST API endpoints. You can use these `cURL` commands directly in your terminal to quickly test the functionality of the MemMachine API.

### 1. Add a Memory Episode (`POST /memory`)

This example demonstrates how to add a new memory episode to both the episodic and profile memory. You'll need to provide a `session` object, the `producer` and `produced_for` IDs, and the `episode_content`.

**Request:**

```
curl --request POST \
  --url [https://memmachine.your-api-domain.com/memory](https://memmachine.your-api-domain.com/memory) \
  --header 'Content-Type: application/json' \
  --data '{
    "session": {
      "user_id": ["test_user_1"],
      "session_id": "session-12345"
    },
    "producer": "test_user_1",
    "produced_for": "agent_1",
    "episode_content": "Hello MemMachine, this is my first memory!",
    "episode_type": "text"
  }'
```

**Expected Response:** A `200 OK` status with a confirmation that the memory episode was added successfully.

### 2. Delete Session Data (`DELETE /memory`)

This example shows how to delete all memory data associated with a specific session. This is useful for cleaning up after a testing session.

**Request:**

```
curl --request DELETE \
  --url [https://memmachine.your-api-domain.com/memory](https://memmachine.your-api-domain.com/memory) \
  --header 'Content-Type: application/json' \
  --data '{
    "session": {
      "session_id": "session-12345"
    }
  }'
```

**Expected Response:** A `200 OK` status indicating the data was successfully deleted.

### 3. Search for Memories (`POST /memory/search`)

Use this example to search for relevant memories within a session. The API will return results from both episodic and profile memory.

**Request:**

```
curl --request POST \
  --url [https://memmachine.your-api-domain.com/memory/search](https://memmachine.your-api-domain.com/memory/search) \
  --header 'Content-Type: application/json' \
  --data '{
    "session": {
      "user_id": ["test_user_1"],
      "session_id": "session-12345"
    },
    "query": "what was my first message?"
  }'
```

**Expected Response:** A `200 OK` status with a JSON object containing the search results.

### 4. Get All Sessions (`GET /sessions`)

This simple endpoint allows you to retrieve a list of all existing sessions in the system.

**Request:**

```
curl --request GET \
  --url [https://memmachine.your-api-domain.com/sessions](https://memmachine.your-api-domain.com/sessions)
```

**Expected Response:** A `200 OK` status with a JSON array of `MemorySession` objects.

### 5. Get Sessions for a Specific User (`GET /sessions/{user_id}`)

To filter sessions by a specific user, use this example by replacing `test_user_1` with your user's ID.

**Request:**

```
curl --request GET \
  --url [https://memmachine.your-api-domain.com/sessions/test_user_1](https://memmachine.your-api-domain.com/sessions/test_user_1)
```

**Expected Response:** A `200 OK` status with a JSON array of `MemorySession` objects associated with `test_user_1`.

### 6. Get Sessions for a Specific Group (`GET /sessions/{group_id}`)

Similarly, this endpoint lets you retrieve sessions belonging to a specific group. Replace `test_group_1` with your group's ID.

**Request:**

```
curl --request GET \
  --url [https://memmachine.your-api-domain.com/sessions/test_group_1](https://memmachine.your-api-domain.com/sessions/test_group_1)
```

**Expected Response:** A `200 OK` status with a JSON array of `MemorySession` objects associated with `test_group_1`.

### 7. Get Sessions for a Specific Agent (`GET /sessions/{agent_id}`)

This example shows how to retrieve sessions for a particular agent by replacing `test_agent_1` with the agent's ID.

**Request:**

```
curl --request GET \
  --url [https://memmachine.your-api-domain.com/sessions/test_agent_1](https://memmachine.your-api-domain.com/sessions/test_agent_1)
```

**Expected Response:** A `200 OK` status with a JSON array of `MemorySession` objects associated with `test_agent_1`.
