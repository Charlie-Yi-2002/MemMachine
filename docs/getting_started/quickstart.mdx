## Installation


### Prerequisites

Before installing MemMachine, ensure you have the following prerequisites:

- **Python 3.12+** - MemMachine requires Python 3.12 or higher
- **Docker & Docker Compose** (optional, for containerized deployment)
- **Neo4j Database** - For graph storage and vector operations
- **PostgreSQL** - For session management
- **OpenAI API Key** - For language models and embeddings

### Method 1: Docker Installation
#### Step 1: Clone and Setup
```bash
git clone https://github.com/MemMachine/MemMachine.git
cd MemMachine
```

#### Step 2: Create Environment File
Create a `.env` file in the project root with your database credentials:
```bash
cat > .env << 'EOF'
NEO4J_USERNAME="neo4j"
NEO4J_PASSWORD="YOUR_NEO4J_PASSWORD"
NEO4J_URI="bolt://localhost:7687"

# Postgres database configuration
POSTGRES_HOST=YOUR_POSTGRES_HOST
POSTGRES_PORT=5432
POSTGRES_USER=YOUR_POSTGRES_USER
POSTGRES_PASS=YOUR_POSTGRES_PASSWORD
POSTGRES_PASSWORD=YOUR_POSTGRES_PASSWORD
POSTGRES_DB=YOUR_POSTGRES_DATABASE

MCP_BASE_URL=http://127.0.0.1:8080
GATEWAY_URL=http://localhost:8080
FAST_MCP_LOG_LEVEL=INFO
MEMORY_CONFIG=/app/configuration.yml
OPENAI_API_KEY="YOUR_VALID_OPENAI_API_KEY"
EOF
```

**Important**: Replace the following placeholders with your actual values:
- `YOUR_NEO4J_PASSWORD` - Your Neo4j password
- `YOUR_POSTGRES_HOST` - Your PostgreSQL host (e.g., AWS RDS endpoint)
- `YOUR_POSTGRES_USER` - Your PostgreSQL username
- `YOUR_POSTGRES_PASSWORD` - Your PostgreSQL password
- `YOUR_POSTGRES_DATABASE` - Your PostgreSQL database name
- `YOUR_VALID_OPENAI_API_KEY` - Your valid OpenAI API key

### Step 3: Create Configuration File
The `configuration.yml` file should contain the following content:

**Important**: Replace the following placeholders with your actual values:
- `YOUR_VALID_OPENAI_API_KEY` - Your valid OpenAI API key (used in multiple places)
- `YOUR_NEO4J_PASSWORD` - Your Neo4j password (must match the password in .env and docker-compose.yml)

```yaml
logging:
  path: /tmp/memory_log
  level: info

long_term_memory:
  derivative_deriver: sentence
  metadata_prefix: "[$timestamp] $producer_id: "
  embedder: my_embedder_id
  reranker: my_reranker_id
  vector_graph_store: my_storage_id

SessionDB:
  uri: sqlitetest.db

Model:
  testmodel:
    model_vendor: openai
    model_name: "gpt-4o-mini"
    api_key: "YOUR_VALID_OPENAI_API_KEY"
    
storage:
  my_storage_id:
    vendor_name: neo4j
    host: memmachine-neo4j-custom
    port: 7687
    user: neo4j
    password: YOUR_NEO4J_PASSWORD

sessionMemory:
  model_name: testmodel
  message_capacity: 500
  max_message_length: 16000
  max_token_num: 8000

embedder:
  my_embedder_id:
    model_name: "text-embedding-3-small"
    api_key: "YOUR_VALID_OPENAI_API_KEY"

reranker:
  my_reranker_id:
    type: "rrf-hybrid"
    reranker_ids:
      - id_ranker_id
      - bm_ranker_id
      - ce_ranker_id
  id_ranker_id:
    type: "identity"
  bm_ranker_id:
    type: "bm25"
  ce_ranker_id:
    type: "cross-encoder"
    model_name: "cross-encoder/qnli-electra-base"
```

**Important**: Replace all instances of `YOUR_VALID_OPENAI_API_KEY_HERE` with your valid OpenAI API key.

#### Step 4: Create Docker Compose File
The `docker-compose.yml` file should contain:

```yaml
services:
  # Neo4j database for graph storage
  neo4j:
    image: neo4j:ubi9
    container_name: memmachine-neo4j-custom
    restart: unless-stopped
    ports:
      - "7474:7474"  # HTTP
      - "7473:7473"  # HTTPS
      - "7687:7687"  # Bolt
    environment:
      NEO4J_EDITION: community
      NEO4J_AUTH: neo4j/YOUR_NEO4J_PASSWORD
      NEO4J_server_bolt_thread__pool__max__size: 2000
      NEO4J_server_memory_heap_initial__size: 512m
      NEO4J_server_memory_heap_max__size: 1G
    volumes:
      - neo4j_data:/data
      - neo4j_logs:/logs
    healthcheck:
      test: ["CMD", "cypher-shell", "-u", "neo4j", "-p", "YOUR_NEO4J_PASSWORD", "RETURN 1"]
      interval: 30s
      timeout: 10s
      retries: 5
      start_period: 30s

volumes:
  neo4j_data:
    driver: local
  neo4j_logs:
    driver: local

networks:
  default:
    name: memmachine-network
```

#### Step 5: Build and Start Services

1. Build the MemMachine Docker image:
```bash
docker build -t memmachine .
```

2. Start Neo4j:
```bash
docker compose up neo4j -d
```
3. PostgreSQL Setup 
For session management, you can use PostgreSQL:

 - **Install PostgreSQL:**
   ```bash
   # Ubuntu/Debian
   sudo apt-get install postgresql postgresql-contrib
   
   # macOS
   brew install postgresql
   
   # Windows
   # Download from postgresql.org
   ```
 -  **Create database:**
   ```sql
   CREATE DATABASE memmachine;
   CREATE USER memmachine WITH PASSWORD 'your_password';
   GRANT ALL PRIVILEGES ON DATABASE memmachine TO memmachine;
   ```

4. Run MemMachine with custom configuration:
```bash
docker run -p 8080:8080 --env-file .env -v $(pwd)/configuration.yml:/app/configuration.yml:ro --network memmachine-network memmachine
```

#### Step 6: Verify Setup

 1. Check Neo4j is running:
```bash
docker ps | grep neo4j
```

2. Test MemMachine API:
```bash
curl http://localhost:8080/docs
```

### Database Configuration Summary
- **PostgreSQL**: Your PostgreSQL database (configured via POSTGRES_HOST in .env)
- **Neo4j**: Local Docker container (localhost:7687)
- **MemMachine API**: http://localhost:8080

### Troubleshooting

#### Common Issues:

1. **Neo4j Connection Error**: Ensure the Neo4j host in `configuration.yml` is set to `memmachine-neo4j-custom` (not `localhost`)

2. **OpenAI API Key Error**: Make sure you have a valid OpenAI API key and it's correctly set in both the `.env` file and `configuration.yml`

3. **Port Access Issues**: If you can't access the API, ensure the MemMachine container is running on the `memmachine-network` and port 8080 is accessible

4. **Container Network Issues**: Make sure both containers are on the same Docker network (`memmachine-network`)

#### Useful Commands:

- Check container status: `docker ps`
- View container logs: `docker logs <container_id>`
- Stop containers: `docker stop <container_id>`
- Remove containers: `docker rm <container_id>`
- Check available API endpoints: `curl -s http://localhost:8080/openapi.json | jq '.paths | keys'

### Add a Memory 

```Python
import requests
import json

# The API endpoint to add memory
url = 'https://memmachine.mintlify.app/llms/add_memmachine'

# The "Hello World" text we want to add
payload = {
    'content': 'Hello, MemMachine! This is my first memory.'
}

# Set up the headers
headers = {
    'Content-Type': 'application/json'
}

# Make the POST request
response = requests.post(url, headers=headers, data=json.dumps(payload))

# Print the response from the server
print(f"Status Code: {response.status_code}")
print(f"Response: {response.text}")
```

This guide provides simple "Hello World" examples for each of the core REST API endpoints. You can use these `cURL` commands directly in your terminal to quickly test the functionality of the MemMachine API.

### 1. Add a Memory Episode (`POST /memory`)

This example demonstrates how to add a new memory episode to both the episodic and profile memory. You'll need to provide a `session` object, the `producer` and `produced_for` IDs, and the `episode_content`.

**Request:**

```
curl --request POST \
  --url [https://memmachine.your-api-domain.com/memory](https://memmachine.your-api-domain.com/memory) \
  --header 'Content-Type: application/json' \
  --data '{
    "session": {
      "user_id": ["test_user_1"],
      "session_id": "session-12345"
    },
    "producer": "test_user_1",
    "produced_for": "agent_1",
    "episode_content": "Hello MemMachine, this is my first memory!",
    "episode_type": "text"
  }'
```

**Expected Response:** A `200 OK` status with a confirmation that the memory episode was added successfully.

### 2. Delete Session Data (`DELETE /memory`)

This example shows how to delete all memory data associated with a specific session. This is useful for cleaning up after a testing session.

**Request:**

```
curl --request DELETE \
  --url [https://memmachine.your-api-domain.com/memory](https://memmachine.your-api-domain.com/memory) \
  --header 'Content-Type: application/json' \
  --data '{
    "session": {
      "session_id": "session-12345"
    }
  }'
```

**Expected Response:** A `200 OK` status indicating the data was successfully deleted.

### 3. Search for Memories (`POST /memory/search`)

Use this example to search for relevant memories within a session. The API will return results from both episodic and profile memory.

**Request:**

```
curl --request POST \
  --url [https://memmachine.your-api-domain.com/memory/search](https://memmachine.your-api-domain.com/memory/search) \
  --header 'Content-Type: application/json' \
  --data '{
    "session": {
      "user_id": ["test_user_1"],
      "session_id": "session-12345"
    },
    "query": "what was my first message?"
  }'
```

**Expected Response:** A `200 OK` status with a JSON object containing the search results.

### 4. Get All Sessions (`GET /sessions`)

This simple endpoint allows you to retrieve a list of all existing sessions in the system.

**Request:**

```
curl --request GET \
  --url [https://memmachine.your-api-domain.com/sessions](https://memmachine.your-api-domain.com/sessions)
```

**Expected Response:** A `200 OK` status with a JSON array of `MemorySession` objects.

### 5. Get Sessions for a Specific User (`GET /sessions/{user_id}`)

To filter sessions by a specific user, use this example by replacing `test_user_1` with your user's ID.

**Request:**

```
curl --request GET \
  --url [https://memmachine.your-api-domain.com/sessions/test_user_1](https://memmachine.your-api-domain.com/sessions/test_user_1)
```

**Expected Response:** A `200 OK` status with a JSON array of `MemorySession` objects associated with `test_user_1`.

### 6. Get Sessions for a Specific Group (`GET /sessions/{group_id}`)

Similarly, this endpoint lets you retrieve sessions belonging to a specific group. Replace `test_group_1` with your group's ID.

**Request:**


**Expected Response:** A `200 OK` status with a JSON array of `MemorySession` objects associated with `test_group_1`.

### 7. Get Sessions for a Specific Agent (`GET /sessions/{agent_id}`)

This example shows how to retrieve sessions for a particular agent by replacing `test_agent_1` with the agent's ID.
